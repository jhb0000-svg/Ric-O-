# 🚀 Phase 1 & 2: 다중 조건(AND/OR) 노출형 고급 필터 파이프라인

**Project:** 멀티 플랫(Flat) 검색 패널 전환 및 다중 조건 연산 도입
**Goal:** 답답한 콤보박스를 없애고, 전문 검색 솔루션처럼 모든 조건 입력창을 상단에 펼친 뒤 "교집합(AND)"과 "합집합(OR)" 그래프 위상 검색 엔진을 구현합니다.

---

## 📌 1. PRD (Product Requirements Document)

### 요구사항 분석
1. **콤보박스 폐지**: 카테고리를 눌러서 하나씩 검색하던 기존 UI를 전면 폐기.
2. **다중 조건 입력 패널(Flat UI)**: 기록물, 기관명, 생산년도, 담당자, 단위과제의 **검색창 5개를 화면 최상단 가로 띠 형태로 나란히 모두 노출**시킵니다.
3. **AND/OR 토글 지원**: 2개 이상의 창에 문자를 입력했을 때 동작 방식을 결정하는 [AND / OR] 라디오버튼(혹은 스위치)을 구축합니다.

### ❓ Clarifying Question (핵심 로직 확인)
**"AND" 검색의 작동 원리**에 대한 동의가 필요합니다.
우리가 구축한 데이터는 RDBMS(표)가 아니라 **관계형 점과 선(그래프)**입니다. 담당자와 기관명은 서로 다른 노드(동그라미)입니다. 
따라서 JS 검색 엔진이 그래프 상의 `AND`를 처리할 때 다음과 같이 작동하는 것이 맞는지 승인해 주시면 됩니다.

* **OR 검색 로직**: [기관명: "보건복지부"], [담당자: "정은경"] 검색 시 -> "보건복지부" 노드와 "정은경" 노드를 **각각** 모두 찾아서 두 파편을 보여주는 합집합 시각화.
* **AND 검색 로직 (고급 관계망 분석)**: "정은경" 이면서 동시에 "보건복지부"에 속한 **'공통 기록물 노드(교집합 허브)'를 선을 따라 역추적하여 발굴**합니다. 즉, 둘 모두에게 직접 선이 연결된 기록물과 그 중심 클러스터만 화면에 독점적으로 띄웁니다.

## 📝 2. 작업 계획 (Task Plan) - *승인 대기 중*

### [ ] Task 1: UI 패널 구조조정 (`templates/index.html`)
- 상단 헤더 아래에 짙은 네이비색 블록의 `<div class="advanced-filter-panel">` 영역을 통째로 삽입합니다.
- `[AND/OR 토글] + 5개의 검색 input(기록물/기관/연도/담당자/업무) + [통합 검색 필터링] 버튼`을 가로 정렬(Flex)로 배치합니다.

### [ ] Task 2: 그래프 토폴로지 역추적 JS 엔진 구현 (OR 연산)
- 자바스크립트에서 입력받은 N개의 검색 필드에 값들을 순회합니다. 
- OR 연산일 경우: 입력값이 존재하는 모든 Node를 찾고, 이들과 연결된 1-Hop 이웃을 모두 모아 화면에 통합 렌더링.

### [ ] Task 3: 그래프 토폴로지 역추적 JS 엔진 구현 (AND 연산)
- AND 연산일 경우: `RecordResource` (기록물) 노드를 메인 척추로 기준 삼습니다.
- JS 로직이 각 기록물의 엣지(Edge)를 타고 주변 1-Hop 노드들을 순회하여, 사용자가 입력한 여러 개의 조건("정은경" AND "2021" AND "보건복지부")을 **선을 타고 들어가 모두 만족하는지 정밀 검사**합니다. 완전 일치하는 중심 기록물 조직망만 화면에 남깁니다.
